# 退款规则

## 1. 文档目的

本文件用于说明智选商城退款申请、退款金额、退款路径、优惠抵扣、运费退还和异常退款处理规则，适用于智能客服在退款咨询和退款工单创建场景中的知识检索。

## 2. 退款适用场景

### 2.1 可申请退款的常见场景

1. 用户已付款但商家尚未发货，可申请取消订单并退款。
2. 用户收到商品后因商品质量问题、错发漏发、运输破损等原因申请退货退款。
3. 用户在规定时间内因个人原因申请无理由退货，并符合商品完好标准。
4. 用户申请售后维修或换货失败，经平台确认后可转为退款处理。
5. 商品长期未发货、物流异常丢件、商家无法履约时，可申请退款或人工介入。

### 2.2 不支持直接退款的常见场景

1. 商品已签收且超过平台规定售后时限，且无法证明存在质量问题。
2. 商品因用户人为损坏、私自拆修、进液、摔落、改装等原因影响正常售后。
3. 用户无法提供订单号、商品凭证或有效问题证明。
4. 商品页面明确标注不支持无理由退货，且商品不存在质量问题。

## 3. 退款金额计算

### 3.1 基本退款金额

1. 退款金额以用户实际支付金额为基础计算，不超过订单实付金额。
2. 如订单包含多件商品，部分退款时按对应商品的实付分摊金额计算。
3. 使用优惠券、满减、积分、会员权益抵扣的订单，退款时按系统分摊结果处理。
4. 若订单存在赠品、组合装、套装优惠，退款时需同时退回赠品或按规则扣除赠品价值。

### 3.2 运费退款规则

1. 因商品质量问题、错发、漏发、运输破损等平台或商家责任导致退款的，平台可退还合理运费。
2. 因用户个人原因发起无理由退货的，发货运费和退回运费是否退还，以订单页面和售后审核结果为准。
3. 包邮订单发生部分退款时，若剩余商品不满足包邮条件，系统可能按规则扣除相应运费。
4. 用户自行选择高价物流、到付物流或未经平台确认的物流方式，额外费用可能无法全额退还。

## 4. 退款路径和到账说明

### 4.1 原路退回原则

1. 退款原则上按用户原支付路径退回，例如银行卡、第三方支付账户、平台余额或分期支付账户。
2. 多种支付方式组合支付的订单，退款将根据支付渠道和系统分摊规则分别退回。
3. 使用礼品卡、余额、积分等支付的部分，一般退回至对应账户或权益账户。

### 4.2 到账时间说明

1. 平台审核通过后会发起退款，具体到账时间受支付渠道、银行处理、节假日等因素影响。
2. 智能客服不得承诺“退款一定今天到账”“马上到账”“百分百到账时间”等绝对表述。
3. 若超过预计时间仍未到账，用户可提供订单号和支付凭证，平台将协助查询退款状态。

## 5. 优惠券和积分退款规则

### 5.1 优惠券退回

1. 未使用或订单全额退款时，仍在有效期内且符合退回条件的优惠券可能退回账户。
2. 已过期优惠券、活动专属券、一次性券、部分特殊补贴券，退款后可能不予退回。
3. 部分退款场景下，优惠券可能按系统分摊后部分失效或不退回。

### 5.2 积分退回

1. 使用积分抵扣的订单发生退款时，积分是否退回以积分规则和系统处理结果为准。
2. 若积分已过有效期或属于活动限时积分，退款后可能无法恢复。
3. 因平台原因导致的退款，客服可建议用户提交人工复核。

## 6. 退款审核流程

1. 用户提交退款申请，填写退款原因并上传必要凭证。
2. 系统根据订单状态、商品状态、售后时限和风险规则进行初审。
3. 如需退货，用户应按平台要求寄回商品并填写物流单号。
4. 平台或商家验收商品后，确认退款、换货、维修或驳回申请。
5. 退款发起后，用户可在订单详情页或客服系统查询进度。

## 7. 异常退款处理

1. 订单号不存在、用户身份不匹配、商品不在订单中时，不允许直接退款。
2. 高金额退款、批量退款、频繁退款、异常账号退款，应进入人工复核。
3. 用户要求绕过平台、私下转账或线下退款时，客服必须拒绝并引导用户走平台流程。
4. 涉及投诉、纠纷、争议证据不足的退款，建议创建工单并转人工客服处理。

## 8. 客服回复规范

1. 客服应使用“预计”“一般”“以审核结果为准”等审慎表述。
2. 客服不得承诺无条件退款、固定到账时间或超出规则的赔付金额。
3. 涉及用户手机号、银行卡号、收货地址等隐私信息时，应进行脱敏展示。
4. 若无法自动判断，智能客服应提示用户提交工单或转人工客服。
