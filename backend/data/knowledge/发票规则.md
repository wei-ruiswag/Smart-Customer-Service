# 发票规则

## 1. 文档目的

本文件用于说明智选商城发票申请、发票类型、开票信息、发票下载、发票修改、退货退款后的发票处理和异常开票处理规则，适用于发票咨询和发票工单创建场景。

## 2. 发票类型

### 2.1 电子普通发票

1. 电子普通发票适用于个人或企业日常报销、售后凭证和财务留存。
2. 用户可在下单时选择开具电子普通发票，也可在订单完成后通过订单详情页申请。
3. 电子普通发票通常支持在线查看、下载和发送至邮箱。
4. 电子普通发票与纸质普通发票在合规用途上具有相应效力，具体报销要求以用户所在单位规定为准。

### 2.2 增值税专用发票

1. 增值税专用发票通常适用于具备相应资质的企业用户。
2. 用户申请专用发票时，应准确填写公司名称、纳税人识别号、注册地址、注册电话、开户行和银行账号等信息。
3. 首次申请专用发票可能需要提交或维护开票资质信息。
4. 专用发票开具后如需修改，通常需要先作废、红冲或按平台规则提交人工处理。

## 3. 开票信息要求

### 3.1 个人发票

1. 个人发票抬头可填写个人姓名或“个人”。
2. 个人发票一般不需要填写纳税人识别号。
3. 用户应确认邮箱、手机号或订单账户可正常接收发票通知。

### 3.2 企业发票

1. 企业发票抬头应与企业工商登记名称一致。
2. 企业发票需填写准确的纳税人识别号。
3. 如开具增值税专用发票，还需补充注册地址、电话、开户行和银行账号。
4. 开票信息错误可能导致发票无法报销或需要重新开具。

## 4. 开票时间和下载方式

1. 已完成订单通常可在订单详情页申请或下载发票。
2. 部分订单需在确认收货、交易完成或商家审核后才能开具发票。
3. 电子发票开具后，用户可在订单详情页、发票中心或邮箱中查看。
4. 若发票长时间未生成，用户可提交发票工单，由客服协助查询。

## 5. 发票内容规则

1. 发票内容应根据实际购买商品或服务开具，不得虚开、错开或随意更改。
2. 手机、耳机、笔记本、家电、配件等实物商品，通常按商品明细或商品类别开具。
3. 用户要求开具与实际交易不一致的发票内容时，客服应拒绝并说明原因。
4. 发票金额应与订单实付金额、退款金额和系统规则保持一致。

## 6. 退货退款后的发票处理

1. 订单全额退款后，已开具发票可能需要作废或红冲。
2. 订单部分退款后，发票金额可能需要根据退款后实际金额重新处理。
3. 用户申请退货退款时，应根据平台要求配合处理已开具发票。
4. 若用户已用于报销，应根据财务和税务要求自行确认后续处理方式。

## 7. 发票修改和补开

1. 订单未开票前，用户可在允许范围内修改发票抬头和开票信息。
2. 发票已开具后，能否修改取决于发票类型、订单状态和平台规则。
3. 因用户填写错误导致重新开票，可能需要用户提交原发票作废或红冲申请。
4. 因平台或商家原因导致错开、漏开，用户可提交工单申请补开或重开。

## 8. 异常开票处理

1. 订单号不存在、订单未完成、交易异常或用户身份不匹配时，不支持直接开票。
2. 发票抬头、税号、邮箱等信息格式错误时，系统可能拒绝开票申请。
3. 高金额订单、企业批量开票、跨月开票、红冲重开等复杂场景，应转人工处理。
4. 用户要求虚开发票、拆分虚假金额、开具非实际商品内容时，客服必须拒绝。

## 9. 客服回复规范

1. 客服应提示用户以订单详情页、发票中心和平台审核结果为准。
2. 不得承诺“立即开票”“一定可改”“一定能报销”等绝对表述。
3. 涉及企业税号、银行账号、注册地址等敏感信息时，应避免在普通对话中完整展示。
4. 发票争议、补开、红冲、专票资质等场景，建议创建发票工单并转人工处理。
